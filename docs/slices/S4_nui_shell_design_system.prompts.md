## Coordination contract

S4 has three agents: Frontend, Integration and QA. Frontend owns the React shell, design system primitives, route placeholders and TypeScript NUI message interfaces. Integration owns FiveM client entrypoints, `ox_target`, keybinds, focus management, NUI callbacks and `fxmanifest.lua`. QA owns visual/browser/in-game smoke and DoD verification.

Suggested order: Frontend and Integration may work in parallel only after both accept the NUI message contract below. Frontend should create `web/src/types/nui.ts` early. Integration must consume that contract exactly when sending messages. QA can prepare smoke procedures immediately, then run final checks after both outputs are integrated.

Baseline NUI contract to preserve unless both agents document a better one in the mini-brief:

```ts
type NuiMode = 'manager' | 'field';

type NuiInboundMessage =
  | { action: 'sonar:farm:nui:open'; payload: { mode: NuiMode; route?: string } }
  | { action: 'sonar:farm:nui:close' };

type NuiCloseCallbackResponse = { ok: true } | { ok: false; error: string };
```

## Interface contracts

Send to NUI:

```lua
SendNUIMessage({
    action = 'sonar:farm:nui:open',
    payload = {
        mode = 'manager', -- or 'field'
        route = '/dashboard'
    }
})
```

Close message/callback:

- Frontend calls callback endpoint for `close`.
- Lua clears focus and sends optional `sonar:farm:nui:close` message back.

Events are not required unless you need cross-resource communication. If added, use `sonar:farm:*` naming and document them.

## Coordination contract

**Why two slices, one bundle.** S10 DoD ("sell batch → payout → batch_id leaves inventory")
cannot close without S9 stash mechanics. Both slices share the `sonar_farm_storage_units` schema,
the `StorageService` API, and the Finance.Credit call. Backend must finish first; Integration
follows; QA runs last.

**Shared source of truth.** Every sub-agent reads `docs/slices/B2_storage_npc_sale.md` §8 before
writing anything. §8 defines: DB schemas (storage_units + storage_contents), all 4 events,
config shapes for `storage.lua` + `npcs.lua`, load order, boot sequence, payout formula, and
ox_inventory stash pattern.

**Hard rules for all sub-agents:**
- No `MySQL.Sync.*`. Only `Sonar.Farm.DB.*` (async await).
- No `exports['other_farm_resource']:...` — only declared `ox_inventory`/`ox_target`/`ox_lib` exports.
- No hardcoded numbers — prices, coords, capacity in config files.
- No hardcoded strings — everything through `locale()`.
- Anti-patterns 1–10 in `.windsurf/rules/05_anti_patterns.md` are absolute.
- Next migration number is **008** (`008_storage.sql`). Do not use 007.

### 4. `server/storage/storage_service.lua`

Expose `Sonar.Farm.StorageService` with methods defined in B2 §4:

**`Boot() -> boolean`**
- For each unit in `Config.Farm.Storage.units`: INSERT OR IGNORE into
  `sonar_farm_storage_units` (config-owned columns only; do not reset if already exists).
- Also registers ox_inventory stash per unit:
  `exports.ox_inventory:RegisterStash('sonar_farm_silo_' .. unit.storage_id, { label=..., slots=..., weight=unit.max_weight_kg*1000 })`.
  Anti-pattern §1 note: `exports.ox_inventory` is a declared dependency — allowed.
- Logs `[storage] Storage boot ready: N units seeded`.

**`Deposit(storage_id, player_cid, batch_id, item_name, metadata) -> ok, error`**
- Validate unit exists (`GetUnit`).
- Validate capacity: query `sonar_farm_storage_contents` count for `storage_id` < `max_slots`.
- Validate `batch_id` not already deposited.
- Insert into `sonar_farm_storage_contents`.
- Emit `sonar:farm:storage:deposited`.
- Return `ok, nil` on success.

**`Withdraw(storage_id, player_cid, batch_id) -> ok, metadata, error`**
- Validate row exists in `sonar_farm_storage_contents`.
- Fetch metadata from `sonar_farm_batches` (join on `batch_id`) — needed for re-adding to player inventory.
- Delete row from `sonar_farm_storage_contents`.
- Emit `sonar:farm:storage:withdrawn`.
- Return `ok, metadata, nil`.

**`GetUnit(storage_id) -> unit | nil`** — single row from DB.
**`ListContents(storage_id) -> content[]`** — rows from `sonar_farm_storage_contents` for given unit.

All DB ops use `Sonar.Farm.DB.*` (async await). No `MySQL.Sync`.

### 5. `server/npcs/npc_buyer_service.lua`

Expose `Sonar.Farm.NPCBuyerService` with methods defined in B2 §4:

**`Boot() -> boolean`**
- Validates `Config.Farm.NPCs.buyers` exists. Logs count.

**`GetBuyer(buyer_id) -> buyer_config | nil`**
- Returns config table for given `buyer_id`.

**`CalculatePayout(buyer_id, crop_type, quality_grade, weight_kg) -> amount, breakdown`**
- Reads buyer config.
- `payout = base_price_per_kg × quality_multipliers[quality_grade] × weight_kg`.
- Also applies `freshness_multiplier` (1.0 stub).
- Returns `amount` (number, rounded to 2 decimal places) and `breakdown` table
  `{ base_price, quality_mult, freshness_mult, weight_kg, total }`.

**`ExecuteSale(src, buyer_id, batch_id) -> ok, payout, error`**
- Validate buyer exists.
- Validate `batch_id` is in player's ox_inventory:
  `local count = exports.ox_inventory:Search(src, 'count', item_name)`.
  If 0 → return false, nil, 'batch_not_in_inventory'.
- Read batch metadata from ox_inventory:
  `local items = exports.ox_inventory:Search(src, 'slots', item_name)` — find slot with matching `batch_id`.
- Validate batch not already sold:
  `SELECT sold_ts FROM sonar_farm_batches WHERE batch_id = ?` — if `sold_ts IS NOT NULL` → reject.
- Calculate payout via `CalculatePayout`.
- Emit `sonar:farm:sale:initiated` payload.
- Execute atomically:
  1. `exports.ox_inventory:RemoveItem(src, item_name, 1, { batch_id = batch_id })`.
  2. `UPDATE sonar_farm_batches SET sold_ts = ? WHERE batch_id = ?`.
  3. `Sonar.Farm.Finance.Credit(src, payout, 'sale:' .. buyer_id .. ':' .. batch_id)`.
- If any step fails: rollback (re-add item if removed, do NOT re-add if Finance.Credit fails —
  log error and document as known limitation; Finance audit table has the record).
- Emit `sonar:farm:sale:completed`.
- Return `true, payout, nil`.

Anti-pattern §1: `exports.ox_inventory` calls are allowed (declared dependency).
Anti-pattern §2: payout is ALWAYS calculated server-side from DB metadata — never trust client values.

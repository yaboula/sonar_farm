## Test focus

### Required behavior

- Finance boot selects adapter and logs it.
- QBox baseline works through `native_bridge` / Bridge.
- Credit increases framework bank balance and writes exactly one successful movement row.
- Replaying same idempotency key does not credit twice.
- Debit decreases framework bank balance and writes exactly one successful movement row.
- Debit above balance fails safely and does not write a successful movement row.
- Adapter selection is visible in logs.
- No direct external bank dependency is required to start `sonar_farm_core`.

### Static checks

Search for forbidden hard dependencies in production code:

- `exports['sonar_bank']`
- `exports['sonar_bank_app']`
- `exports['Renewed-Banking']`
- `exports['okokBanking']`
- `exports['qs-` or similar

These are only acceptable inside optional future adapter files if explicitly config-gated and documented. In S3 baseline they should not appear.

## Coordination contract

**Why three slices, one bundle.** S13/S14/S15 all activate quality factor stubs that were
neutral (default score) since B1. They share `sonar_farm_quality_tracking` as the single
persistence target, the same `track()`/`get()` interface already coded in B1, and the same
`ox_target → anim → server callback → factor update` client pattern. Backend implements all
three factor tracker upgrades in one pass; Integration wires three parallel client files;
QA validates the full quality decision surface in a single smoke.

**Shared source of truth.** Every sub-agent reads `docs/slices/B3_factor_activation.md` §8
before writing anything. §8 defines: factor tracker API extensions, all 5 events, PestService
FSM, migration `011_pest_severity.sql`, item config shape, scheduler integration point,
lib.callback registrations, and pest visual approach.

**Hard rules for all sub-agents:**
- No `MySQL.Sync.*` — only `Sonar.Farm.DB.*` (async await).
- No framework-specific calls — bridge only for money/player operations.
- No hardcoded numbers — all thresholds, probabilities, durations in config.
- No hardcoded strings — everything through `locale()`.
- Anti-patterns 1–10 in `.windsurf/rules/05_anti_patterns.md` are absolute.
- Factor tracker methods are **additive** — do NOT replace existing `track()`/`get()` from B1.
- Next migration number is **011** (`011_pest_severity.sql`). Do not use 010.

### 7. Upgrade `server/quality/factors/pest.lua`

Add three methods to existing `PestFactor` table:

**`PestFactor.Appear(plot_id, pest_type)`**
- Validate no active pest already (GetStatus returns nil or 'none').
- `Sonar.Farm.DB.execute` to set `pest_detected_ts = UNIX_TIMESTAMP()`, `pest_severity = 'detected'`.
- Call `PestFactor:track(plot_id, 'pest_appeared', { score = current })` (score unchanged on appear).
- Emit `sonar:farm:pest:appeared`.
- Return ok, nil.

**`PestFactor.Treat(plot_id, pesticide_item)`**
- Read current `pest_severity`. If 'none' → reject.
- Get crop_type from `sonar_farm_crops` for this plot_id.
- Get crop pest config. Determine restoration amount:
  - `pest_severity = 'detected'` → `treat_score_restore` points up from current.
  - `pest_severity = 'severe'` → restore to `severe_treat_ceiling` (ceiling, not delta).
- Call `PestFactor:track(plot_id, 'pest_treated', { score = new_score })`.
- `Sonar.Farm.DB.execute` to set `pest_detected_ts = NULL`, `pest_severity = 'none'`.
- Emit `sonar:farm:pest:treated`.
- Return ok, new_score, nil.

**`PestFactor.GetStatus(plot_id)`**
- Query `SELECT pest_detected_ts, pest_severity FROM sonar_farm_quality_tracking WHERE plot_id = ?`.
- If no row or `pest_severity = 'none'`: return nil.
- Return `{ severity = row.pest_severity, detected_ts = row.pest_detected_ts }`.

### 8. `server/pests/pest_service.lua`

Expose `Sonar.Farm.PestService`:

**`Boot()`**
- Validates `Config.Farm.Crops` exists. Logs pest-susceptible crop count.

**`TickCrops(active_crops)`**
- Called from `scheduler.lua` on each tick.
- `active_crops` is the array already fetched by the scheduler (crop rows with `plot_id`,
  `crop_type`, `stage`).
- For each crop where `stage >= 1 AND stage <= 3`:
  - Skip if plot already has active pest (`PestFactor.GetStatus(plot_id) ~= nil`).
  - Read `Config.Farm.Crops[crop_type].pests.spawn_probability_per_tick`.
  - `if math.random() < probability` → pick random pest from `susceptible_to` list →
    call `PestFactor.Appear(plot_id, pest_type)`.
- For each crop with active pest (query `sonar_farm_quality_tracking` for non-null `pest_detected_ts`
  and `pest_severity = 'detected'`):
  - Check `(UNIX_TIMESTAMP() - pest_detected_ts) / 3600 >= severity_hours`.
  - If yes → update `pest_severity = 'severe'` → emit `sonar:farm:pest:severe`.

**`GetActivePests()`** — returns all rows from `sonar_farm_quality_tracking` where
`pest_severity != 'none'` — used by client to sync visual state on resource start.

### 10. Tests

**`irrigation_spec.lua`** (minimum 5 cases):
- Optimal water (1 charge) → score increases by `score_gain_per_water`.
- Water 3+ times same stage → overwater penalty applied, floor respected.
- Score never exceeds `optimal_ceiling`.
- Score never goes below 0.
- Application count cache resets on stage advance event.

**`fertilization_spec.lua`** (minimum 5 cases):
- Correct fertilizer → score up to ceiling.
- Wrong fertilizer → `wrong_penalty` applied.
- Over-fertilize (2nd application same stage) → floor reached.
- Score never below `overfertilize_floor` from over-fertilize.
- Different crop_type reads its own config (corn vs wheat optimal_items differ).

**`pest_spec.lua`** (minimum 6 cases):
- Appear stamps `pest_detected_ts` and `pest_severity = 'detected'`.
- Appear when pest already active → rejected.
- Treat detected pest → score restored, `pest_detected_ts = NULL`.
- Treat severe pest → score capped at `severe_treat_ceiling`.
- TickCrops probability=1.0 → pest appears.
- TickCrops severity timer expired → `pest_severity` becomes 'severe'.
